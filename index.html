Logic Explanation

The app uses a single array called lootItems declared at the top of script.js, outside all functions, so it persists across button clicks throughout the session. Each time the user clicks "Add Loot," the addLoot() function runs validation checks using if statements: it checks that the loot name is not empty, that the loot value is a valid number, and that it is not negative. If validation passes, a new object with a name and value property is pushed into the array.

After each addition, renderLoot() uses a traditional for loop to iterate through the lootItems array, building the loot list in the DOM item by item, and accumulating a running total as it goes. This total updates live after each item is added.

When the user clicks "Split Loot," the splitLoot() function first validates that the party size is 1 or greater and that there is at least one loot item in the array. If those conditions are met, it divides the total loot by the party size and displays both the total and the per-member share, formatted to two decimal places using toFixed(2).
