Logic Explanation

In Phase 2, my loot array is the single source of truth for the app. Every loot object now also stores a quantity, so the total value for each item is calculated as value times quantity instead of just value alone. I added quantity because real loot often comes in stacks, like 7 daggers instead of just 1, and the totals need to reflect that.

All calculations and rendering now happen inside one function called updateUI(). This function recalculates the total loot using a for loop, rebuilds the loot list rows in the DOM, calculates the per member split, and decides whether the Split button should be enabled or disabled. No other part of my code performs calculations on its own. addLoot() and removeLoot() only change the array and then call updateUI() so the screen always matches the array.

State controls what is visible. If the loot array is empty, the "No loot added" message shows and the totals and results are hidden using a CSS hidden class. If the party size is invalid or there is no loot, the Split button is disabled using the disabled attribute. Conditional logic inside addLoot() prevents empty names, invalid values, negative values, and invalid quantities from ever entering the array, which protects the state before anything is calculated.
