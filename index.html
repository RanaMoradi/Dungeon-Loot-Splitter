Logic Explanation

In this app, I used an array called lootItems to store all the loot that gets added during the game. I declared it at the top of my script.js file, outside of any functions, so that it doesn't reset every time a button is clicked.

When the user clicks Add Loot, my addLoot() function checks that the name isn't empty and that the value is a real number and not negative. If everything looks good, it creates an object with a name and value and pushes it into the array. Then renderLoot() runs a for loop to go through the whole array and display each item on the page, adding up the total as it goes.

When the user clicks Split Loot, my splitLoot() function checks that a party size was entered and that there is actually loot in the array. If both are true, it loops through the array again to get the total, then divides it by the number of party members and shows both the total and each member's share on the page.
